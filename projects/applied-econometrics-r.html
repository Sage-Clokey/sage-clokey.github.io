Final Project

"The Market as Tissue: A Multi-Scale Living Systems Analysis of Informal Price Formation in Lagos"

ML prediction of vendor-quoted rice prices in Lagos informal markets. Outcome variable is price-per-unit-weight quoted to buyers; covariates include buyer socioeconomic presentation (high/low class dress) and transaction context. Applies the Living Systems framework — markets as adaptive networks, not machines — to a causal inference problem. Uses Lagos micro-price dataset.

Also includes spiral_steward_extended.Rmd: extended Spiral Steward econometric analysis connecting living systems theory to market dynamics.

# ============================================================
# Homework 3
# Causal Inference and ML Prediction of 401k Savings Effects
# Sage Clokey
# 3/9/2026
# ============================================================

library(glmnetUtils)
library(randomForest)

# ============================================================
# Load Data
# ============================================================

data <- read.csv("sipp1991.csv")

# ============================================================
# Problem 1: OLS — net_tfa ~ age + inc + e401
# ============================================================

ols1 <- lm(net_tfa ~ age + inc + e401, data = data)
summary(ols1)

# 1a) Interpretation of e401 coefficient (assuming causality):
#
# The coefficient on e401 represents the average increase in net total
# financial assets (in dollars) from an employer offering a 401k plan,
# holding age and income constant. If interpreted causally, it tells us
# how much total savings increases purely due to 401k availability —
# i.e., offering a 401k raises individual savings by approximately [coef]
# dollars on average, controlling for age and income.

# 1b) Why e401 is not causally identified:
#
# The offer of a 401k is not randomly assigned across employers. Firms that
# offer 401k plans tend to be larger, higher-quality employers that also pay
# more and attract workers with stronger preferences for financial planning
# and savings. Even after controlling for age and income, unobserved variables
# such as savings preferences, financial literacy, and firm quality remain
# correlated with both e401 and net_tfa. This omitted variable bias means
# the OLS coefficient on e401 likely overstates the causal effect of 401k
# availability on savings.

# 1c) How the "job choice based on pay" argument helps:
#
# If workers in the 1990s chose jobs primarily based on salary and did not
# consider 401k availability, then conditional on age and income (proxies
# for pay), whether an employer happens to offer a 401k is effectively
# random with respect to worker characteristics like savings preferences.
# This makes e401 conditionally independent of the error term, satisfying
# the assumption needed to give the OLS coefficient a causal interpretation.
# In other words, it rules out the self-selection bias described in part b).

# 1d) Why the argument fails for p401 (enrollment):
#
# Even if workers do not choose jobs based on 401k availability, once offered
# a 401k, they actively choose whether to enroll. Workers who opt in likely
# have stronger savings preferences than those who decline. This self-selection
# means p401 is correlated with unobserved savings preferences — even within
# the same age-income cell — violating conditional independence. The "people
# don't notice 401k offers" argument applies only to the employer's decision
# to offer (e401), not to the worker's voluntary decision to enroll (p401).

# ============================================================
# Problem 2: Post-Lasso
# ============================================================

# 2a) Age coefficient from Problem 1 and plausibility

age_coef <- coef(ols1)["age"]
cat("Coefficient on age:", age_coef, "\n")

# A one-year increase in age is associated with a change of [age_coef] dollars
# in net financial assets, holding income and e401 fixed. For older individuals
# (e.g., nearing retirement), a constant linear effect is implausible: savings
# rates typically accelerate as retirement approaches, then fall afterward.
# The linear model cannot capture this curvature. We also cannot interpret
# this as a causal effect because age is not randomly assigned and is
# correlated with many omitted factors (work experience, cohort effects,
# life-stage savings patterns).

# 2b) Post-lasso Steps 1 and 2 — Lasso with 3-fold CV

# Controls: factor(age), factor(educ), poly(inc,3), poly(fsize,3),
# db, marr, twoearn, pira, hown, and all pairwise interactions via (...)^2.
# p401 is excluded as instructed; e401 is the treatment, not a control.

controls_vars <- "factor(age) + factor(educ) + poly(inc, 3, raw=TRUE) +
                  poly(fsize, 3, raw=TRUE) + db + marr + twoearn + pira + hown"

controls_rhs <- paste0("(", controls_vars, ")^2")

formula_Y <- as.formula(paste("net_tfa ~", controls_rhs))
formula_D <- as.formula(paste("e401 ~",   controls_rhs))

# Step 1: Lasso Y (outcome) on controls
set.seed(1)
lasso_Y <- cv.glmnet(formula_Y, data = data, alpha = 1, nfolds = 3)

# Step 2: Lasso D (treatment) on controls
set.seed(1)
lasso_D <- cv.glmnet(formula_D, data = data, alpha = 1, nfolds = 3)

# 2c) Step 3: Union of selected controls

coef_Y <- coef(lasso_Y, s = "lambda.min")
coef_D <- coef(lasso_D, s = "lambda.min")

vars_Y <- rownames(coef_Y)[coef_Y[, 1] != 0 & rownames(coef_Y) != "(Intercept)"]
vars_D <- rownames(coef_D)[coef_D[, 1] != 0 & rownames(coef_D) != "(Intercept)"]

selected_vars <- union(vars_Y, vars_D)

n_total    <- nrow(coef_Y) - 1
n_selected <- length(selected_vars)

cat("Total potential control variables:", n_total, "\n")
cat("Controls selected by post-lasso:", n_selected, "\n")

# Running OLS with all potential controls would be a bad idea: the number of
# controls likely approaches or exceeds the sample size, making OLS estimates
# numerically unstable or undefined due to perfect multicollinearity. Post-lasso
# selects a parsimonious subset, avoiding overfitting and preserving valid
# inference in the final OLS step.

# 2d) Step 4: Post-lasso OLS

# Build full model matrix from controls formula; extract selected columns
X_mat <- model.matrix(formula_Y, data = data)[, -1]  # drop intercept column

selected_in_mat <- intersect(selected_vars, colnames(X_mat))

df_postlasso <- data.frame(
  net_tfa = data$net_tfa,
  e401    = data$e401,
  X_mat[, selected_in_mat, drop = FALSE]
)

ols_postlasso <- lm(net_tfa ~ ., data = df_postlasso)
summary(ols_postlasso)

cat("\nPost-lasso e401 coefficient:", coef(ols_postlasso)["e401"], "\n")
cat("Simple OLS e401 coefficient:", coef(ols1)["e401"], "\n")

# With richer controls, the post-lasso estimate is more credible. If it is
# smaller than Problem 1's estimate, the simple OLS was upward-biased by
# uncontrolled confounders correlated with both e401 and net_tfa.

# 2e) Counterfactual: average savings if all employers offered 401k

df_cf      <- df_postlasso
df_cf$e401 <- 1

pred_cf <- predict(ols_postlasso, newdata = df_cf)

cat("\nActual mean net_tfa:", mean(data$net_tfa), "\n")
cat("Counterfactual mean net_tfa (all e401=1):", mean(pred_cf), "\n")

# In a linear model, mean(pred_cf) - mean(df_postlasso$net_tfa)
# = beta_e401 * (1 - mean(e401)), the average treatment effect scaled by
# the share of workers currently not offered a 401k.

# 2f) Validity of part d) if controls are not valid for causal inference:
#
# No. Post-lasso is a statistical tool for selecting relevant controls; it
# does not create identification on its own. If the selected controls do not
# make e401 conditionally independent of the error term (e.g., if unobserved
# savings preferences remain correlated with e401 after conditioning), the
# step 4 estimate is still biased. Causal validity depends entirely on the
# economic argument in part 1c — not on the number or method of control
# selection.

# ============================================================
# Problem 3: ML Prediction of net_tfa
# ============================================================

# 3a) Random Forest: 200 trees, minimum leaf size (nodesize) = 10

set.seed(1)
rf_model <- randomForest(net_tfa ~ ., data = data, ntree = 200, nodesize = 10)

oos_r2_rf <- tail(rf_model$rsq, 1)   # OOB R² after all 200 trees (built-in)
cat("\nRandom Forest OOS R² (OOB):", oos_r2_rf, "\n")
print(rf_model)

# 3b) Lasso for prediction: same controls as post-lasso, interacted with e401 and p401

formula_pred <- as.formula(
  paste("net_tfa ~ (e401 + p401 +", controls_vars, ")^2")
)

set.seed(1)
lasso_pred <- cv.glmnet(formula_pred, data = data, alpha = 1, nfolds = 5)

# OOS R²: 1 - (CV MSE at lambda.min) / Var(Y)
oos_r2_lasso <- 1 - min(lasso_pred$cvm) / var(data$net_tfa)
cat("Lasso OOS R²:", oos_r2_lasso, "\n")

cat("\nComparison:\n")
cat("  Random Forest OOS R²:", oos_r2_rf, "\n")
cat("  Lasso OOS R²:", oos_r2_lasso, "\n")

# Random forest does not require specifying a functional form and automatically
# captures complex nonlinear and interaction effects. If it outperforms lasso,
# this suggests meaningful nonlinearities in the relationship between covariates
# and savings that lasso's basis-function approach does not fully capture.

# ============================================================
# Homework Bonus
# Predicting NYPD Stop-and-Frisk Outcomes Using Logit and Lasso
# Sage Clokey
# 3/3/2026

library(glmnetUtils)

# ============================================================
# Problem 1: Data Cleaning
# ============================================================

# 1a) Load data and drop unwanted variables
data <- read.csv("NY_stop_frisk.csv")

pf_vars <- names(data)[startsWith(names(data), "PF_")]
unwanted_vars <- c("DOBNUM", "OFFCODE", "SEARCHED", pf_vars)
data <- data[, !(names(data) %in% unwanted_vars)]

# 1b) Drop observations where OFFUNIF == -9 or EYECOLOR == 'DF'
data <- data[data$OFFUNIF != -9 & data$EYECOLOR != "DF", ]

# ============================================================
# Problem 2: Convert Categorical Variables to Factors
# ============================================================

# 2a) Create vector of column indices for categorical variables
# Based on the codebook: BORO, DAYSTOP2, TIMESTP2, INOUT, TRHSLOC,
# TYPEOFID, SEX, RACE, HAIRCOLR, EYECOLOR, BUILD are categorical.
# Do NOT convert binary (0/1) variables or continuous numeric variables (AGE, etc.)

cat_vars <- c("BORO", "DAYSTOP2", "TIMESTP2", "INOUT", "TRHSLOC",
              "TYPEOFID", "SEX", "RACE", "HAIRCOLR", "EYECOLOR", "BUILD")

cat_vars <- cat_vars[cat_vars %in% names(data)]  # keep only those present
cat_indices <- which(names(data) %in% cat_vars)

# 2b) Loop over indices to convert each column to a factor
for (i in cat_indices) {
  data[, i] <- factor(data[, i])
}

# ============================================================
# Problem 3: 5-Fold Cross-Validation Logit (Plain)
# ============================================================

set.seed(1)

n <- nrow(data)
k <- 5
folds <- sample(rep(1:k, length.out = n))

total_deviance_logit <- 0

for (fold in 1:k) {
  train <- data[folds != fold, ]
  test  <- data[folds == fold, ]

  model <- glm(FRISKED ~ ., data = train, family = binomial)

  pred <- predict(model, newdata = test, type = "response")
  pred <- pmax(pmin(pred, 1 - 1e-10), 1e-10)  # clip to avoid log(0)

  y <- test$FRISKED
  total_deviance_logit <- total_deviance_logit +
    (-2 * sum(y * log(pred) + (1 - y) * log(1 - pred)))
}

cat("CV OOS Deviance (Logit):", total_deviance_logit, "\n")

# ============================================================
# Problem 4: Lasso Logit
# ============================================================

# 4a) Lasso with 5-fold CV using glmnetUtils regression syntax
set.seed(1)

lasso_cv <- cv.glmnet(FRISKED ~ ., data = data, family = "binomial",
                      alpha = 1, nfolds = 5)

lambda_opt <- lasso_cv$lambda.min
cat("Optimal lambda:", lambda_opt, "\n")

# 4b) 1x2 plot: regularization path and CV error vs log(lambda)
par(mfrow = c(1, 2))
plot(lasso_cv$glmnet.fit, xvar = "lambda",
     main = "Lasso Regularization Path")
abline(v = log(lambda_opt), col = "red", lty = 2)

plot(lasso_cv, main = "CV Error vs log(lambda)")

# The optimal penalty (red dashed line) appears in-between the extremes,
# suggesting moderate regularization — neither the full model nor a near-empty one.

# 4c) Total regressors vs. nonzero regressors under optimal penalty
coef_lasso <- coef(lasso_cv, s = "lambda.min")

n_total   <- nrow(coef_lasso) - 1          # exclude intercept
n_nonzero <- sum(coef_lasso[-1, ] != 0)

cat("Total regressors:", n_total, "\n")
cat("Nonzero regressors chosen by lasso:", n_nonzero, "\n")

# If n_nonzero is much smaller than n_total, the model has relatively LOW complexity.

# 4d) Sort nonzero coefficients by magnitude, print top 20
coef_vec     <- coef_lasso[-1, 1]                      # drop intercept, make named vector
nonzero_coefs <- coef_vec[coef_vec != 0]               # keep nonzero
sorted_idx   <- order(abs(nonzero_coefs), decreasing = TRUE)
top20        <- nonzero_coefs[sorted_idx][1:min(20, length(nonzero_coefs))]
print(top20)

# Strong predictors (large |coef|) are likely RF_* (reasons for frisk) and
# CS_* (circumstances of stop) variables. Signs should make intuitive sense:
# e.g., RF_VCRIM (violent crime suspected) should have a positive coefficient.

# 4e) Effect of largest-coefficient variable on odds of being frisked
top_var  <- names(top20)[1]
top_coef <- nonzero_coefs[top_var]
odds_mult <- exp(top_coef)

cat("\nVariable with largest |coefficient|:", top_var, "\n")
cat("Coefficient:", top_coef, "\n")
cat("Odds multiplier (exp(coef)):", odds_mult, "\n")

# Interpretation: a one-unit change in [top_var] multiplies the odds of being
# frisked by exp(coef). If this is a 0/1 variable, the effect is the odds ratio
# for having that characteristic vs. not. A large multiplier (e.g., >5 or <0.2)
# indicates a substantial effect.

# ============================================================
# Problem 5: Predictive Performance
# ============================================================

# 5a) Predicted probabilities histogram
par(mfrow = c(1, 1))
pred_all <- as.vector(predict(lasso_cv, newdata = data,
                               s = "lambda.min", type = "response"))
hist(pred_all, breaks = 30,
     main = "Predicted Probabilities of Being Frisked",
     xlab = "Predicted Probability", col = "steelblue", border = "white")

# If most predictions cluster near 0 or 1, the algorithm is highly certain —
# suggesting possible overfitting. If spread across [0,1], it is more uncertain.

# 5b) CV OOS deviance comparison and OOS R^2
cv_deviance_lasso <- min(lasso_cv$cvm) * n  # total OOS deviance across all obs

# Null model deviance (intercept-only: always predicts p_bar)
p_bar       <- mean(data$FRISKED)
null_dev    <- -2 * sum(data$FRISKED * log(p_bar) +
                          (1 - data$FRISKED) * log(1 - p_bar))

oos_r2 <- 1 - cv_deviance_lasso / null_dev

cat("\nCV OOS Deviance (Logit):", total_deviance_logit, "\n")
cat("CV OOS Deviance (Lasso):", cv_deviance_lasso, "\n")
cat("OOS R^2 (Lasso):", oos_r2, "\n")

# If OOS R^2 is close to 0, the algorithm predicts poorly relative to just
# guessing the mean. If close to 1, it predicts well.

# ============================================================
# Problem 6: Policy Discussion
# ============================================================

# 6a) Representativeness assumption for predicting frisking by race next year:
#
# The representativeness assumption is implausible, but the deeper problem is
# that the entire stop-and-frisk program violates the non-aggression principle.
# Every observation in this dataset represents a person who was physically
# detained by the state without having harmed anyone or damaged anyone's
# property. The data does not measure dangerousness — it measures how often
# the government initiated force against innocent people. Projecting this
# forward by race compounds the injustice: it assumes the state will continue
# to coercively stop individuals based on characteristics they did not choose,
# in neighborhoods they happen to live in. Even if the statistical model were
# perfectly representative, it would still be predicting the perpetuation of
# rights violations. The correct libertarian answer is not to build a better
# predictive model of who gets frisked — it is to abolish the policy entirely,
# since no individual should be subjected to an involuntary physical search
# absent probable cause of an actual crime against another person.

# 6b) Problems with using the algorithm to "objectively" decide whether to frisk:
#
# Dressing up coercion in the language of algorithms does not make it voluntary
# or just. The core problem is not technical — it is moral. Under the non-
# aggression principle, the state has no legitimate authority to physically
# search a person who has not initiated force or fraud against another. An
# 80% predicted probability does not constitute evidence of an actual crime;
# it is a statistical guess that punishes individuals for who they are rather
# than what they have done. Several practical failures compound this:
#
#   - The model was trained on officer behavior, not actual threats, so it
#     encodes and automates existing patterns of state aggression rather than
#     identifying genuine danger.
#   - Lasso selects variables like location, time, race, and clothing — none
#     of which represent a rights violation by the suspect. Using them to
#     justify a search treats people as guilty by association with a profile.
#   - Calling the algorithm "objective" obscures that a human chose what data
#     to collect, what outcome to predict, and what threshold to enforce.
#     The state does not become neutral by outsourcing its decisions to a model.
#   - Individuals who know the algorithm exists will rationally alter their
#     behavior to avoid triggering it — not because they are dangerous, but
#     because they wish to exercise their natural right to be left alone.
#
# The only legitimate use of predictive tools by law enforcement would be in
# voluntary, consensual contexts — such as a private property owner choosing
# who may enter their premises. Applied to public stops by armed agents of the
# state, this algorithm is a machine for systematizing the violation of
# individual liberty.

# ============================================================
# HW2B - NBA Salary Prediction: Lasso and Ridge Regression
# ============================================================

# ---- Setup: Code copied from HW2A Problems 1c, 1d, 2, and 4a ----

# Load the dataset
nba <- read.csv("NBA.csv")

# HW2A Problem 1c: Create dataframe excluding all rows with missing data
nba_omit <- na.omit(nba)

# HW2A Problem 1d: Create nba_imputed, replacing NAs with column means
nba_imputed <- nba
for (i in 9:ncol(nba_imputed)) {
  missing_rows <- is.na(nba_imputed[, i])
  nba_imputed[missing_rows, i] <- mean(nba_imputed[, i], na.rm = TRUE)
}

# HW2A Problem 2: Convert categorical variables to factors
nba_imputed$NBA_Country <- as.factor(nba_imputed$NBA_Country)
nba_imputed$Tm <- as.factor(nba_imputed$Tm)

# HW2A Problem 4a: Restrict to USA players (excluding territories), drop NBA_Country
nba_usa <- nba_imputed[nba_imputed$NBA_Country == "USA", ]
nba_usa <- nba_usa[, -which(names(nba_usa) == "NBA_Country")]

# ============================================================
# HW2B Problems
# ============================================================

library(glmnet)

# Build design matrix and response vector for all subsequent problems
# Exclude Player (column 1); log-transform Salary as response
X <- model.matrix(log(Salary) ~ . - Player, data = nba_usa)[, -1]
y <- log(nba_usa$Salary)

# ============================================================
# Problem 1: Lasso Regression
# ============================================================

# 1a: Run lasso of log(Salary) on all variables except Player name;
#     plot the regularization path
lasso_model <- glmnet(X, y, alpha = 1)
plot(lasso_model, xvar = "lambda", label = TRUE)
title("Problem 1a: Lasso Regularization Path")

# 1b: As we move left to right (increasing lambda), estimated coefficients shrink
#     toward zero; lasso performs variable selection by setting some to exactly zero.
#     At the leftmost point (lambda near 0) coefficients resemble OLS;
#     at the rightmost (large lambda) most are zero. The magnitudes can differ
#     substantially between the two extremes, reflecting how strongly regularization
#     shrinks the estimates.

# 1c: Higher lambda => greater regularization => higher bias, lower variance.
#     Lower lambda => less regularization => lower bias, higher variance.
#     The optimal lambda balances these two forces.

# ============================================================
# Problem 2: 5-Fold Cross-Validation for Lasso
# ============================================================

set.seed(0)
cv_lasso <- cv.glmnet(X, y, alpha = 1, nfolds = 5)
plot(cv_lasso)
title("Problem 2: Lasso 5-Fold CV Error vs. Lambda")

# The CV error is high for large lambda (too much bias from over-regularization)
# and also rises for very small lambda (too much variance from near-OLS fit).
# The U-shape reflects the bias-variance trade-off: the minimum identifies the
# lambda that best balances the two.

# ============================================================
# Problem 3: Extract Lasso Coefficients at Best Lambda
# ============================================================

lasso_coef <- coef(cv_lasso, s = "lambda.min")

cat("\n--- Problem 3: Nonzero Lasso Coefficients at lambda.min ---\n")
nonzero_mask  <- as.vector(lasso_coef) != 0
nonzero_names <- rownames(lasso_coef)[nonzero_mask]
nonzero_vals  <- as.vector(lasso_coef)[nonzero_mask]
print(data.frame(Variable = nonzero_names, Coefficient = nonzero_vals))

cat("\nCoefficients for key fan stats (TRB, AST, STL, BLK, BPM):\n")
for (stat in c("TRB.", "AST.", "STL.", "BLK.", "BPM")) {
  cat(sprintf("  %-6s : %f\n", stat, lasso_coef[stat, 1]))
}
# If these coefficients are zero or near-zero, lasso suggests they are not
# important determinants of salary once other variables are controlled for.

# ============================================================
# Problem 4: Percentage Change in Salary per Unit Increase in Draft Number
# ============================================================

draft_coef <- lasso_coef["NBA_DraftNumber", 1]
pct_change  <- (exp(draft_coef) - 1) * 100

cat("\n--- Problem 4: Draft Number Coefficient ---\n")
cat("Lasso coefficient for NBA_DraftNumber:", draft_coef, "\n")
cat("Percentage change in salary per 1-unit increase in draft number:",
    round(pct_change, 4), "%\n")
# A one-pick increase in draft number (worse draft position) is associated with
# a pct_change% change in salary. Since higher draft numbers = worse picks,
# a negative coefficient indicates lower-drafted players earn less, which is
# unsurprising given draft status correlates with perceived talent.

# ============================================================
# Problem 5: Ridge Regression (5-Fold CV)
# ============================================================

set.seed(0)
cv_ridge <- cv.glmnet(X, y, alpha = 0, nfolds = 5)
plot(cv_ridge)
title("Problem 5: Ridge 5-Fold CV Error vs. Lambda")

ridge_coef <- coef(cv_ridge, s = "lambda.min")
cat("\n--- Problem 5: Ridge Coefficients at lambda.min ---\n")
print(as.matrix(ridge_coef))

# Comparison with lasso: ridge shrinks all coefficients toward zero but keeps
# all nonzero, while lasso performs exact variable selection (some coefs = 0).
# Ridge tends to distribute shrinkage more evenly across correlated predictors.

# ============================================================
# Problem 6: CV OOS R² for Lasso and Ridge
# ============================================================

# OOS R² = 1 - CV_MSE / Var(y)
# cv.glmnet's cvm reports mean squared error across CV folds
var_y          <- var(y)
lasso_oos_r2   <- 1 - min(cv_lasso$cvm) / var_y
ridge_oos_r2   <- 1 - min(cv_ridge$cvm) / var_y

cat("\n--- Problem 6: Cross-Validation OOS R² ---\n")
cat("Lasso CV OOS R²:", round(lasso_oos_r2, 4), "\n")
cat("Ridge CV OOS R²:", round(ridge_oos_r2, 4), "\n")
# Compare these with the IS and OOS R² from OLS in HW2A.
# Regularized models (lasso/ridge) typically have better OOS R² than OLS
# because they reduce overfitting by shrinking coefficients.

# ============================================================
# Problem 7: Lasso with Team Interactions
# ============================================================

# 7a: Story for interacting Tm with another regressor (e.g., USG):
#     Usage rate (USG) may reward different players differently depending on
#     team strategy; a high-usage player on a star-driven team might be paid
#     more than the same usage rate on a rebuilding team. Interacting Tm with
#     USG captures this heterogeneous relationship.

# 7b: Build interaction design matrix: Tm x (all other regressors)
X_interact <- model.matrix(log(Salary) ~ Tm * (. - Player - Tm),
                           data = nba_usa)[, -1]

cat("\n--- Problem 7b: Interaction Model ---\n")
cat("Number of covariates (with team interactions):", ncol(X_interact), "\n")
cat("Number of observations:", nrow(X_interact), "\n")
cat("OLS possible?",
    ifelse(nrow(X_interact) > ncol(X_interact), "Yes", "No — too many covariates"),
    "\n")
# With more covariates than observations, OLS is infeasible (underdetermined system).
# Lasso can still handle this by performing variable selection.

set.seed(0)
cv_lasso_interact <- cv.glmnet(X_interact, y, alpha = 1, nfolds = 5)

# 7c: OOS R² for interaction lasso
lasso_interact_r2 <- 1 - min(cv_lasso_interact$cvm) / var_y
cat("\n--- Problem 7c: CV OOS R² with Interactions ---\n")
cat("Lasso (with team interactions) CV OOS R²:", round(lasso_interact_r2, 4), "\n")
# Compare with Problem 6: adding interactions may improve OOS R² if team context
# genuinely moderates the relationship between player stats and salary.

# ============================================================
# Problem 8: Forecast LeBron James's Salary
# ============================================================

cat("\n--- Problem 8: LeBron James Salary Forecast ---\n")
lebron_idx <- which(nba_usa$Player == "LeBron James")

if (length(lebron_idx) > 0) {
  # LeBron is in the training data, so we can index directly into X_interact
  lebron_X_interact <- X_interact[lebron_idx, , drop = FALSE]
  lebron_pred_log   <- as.numeric(predict(cv_lasso_interact,
                                          newx = lebron_X_interact,
                                          s    = "lambda.min"))
  lebron_actual_log <- log(nba_usa$Salary[lebron_idx])

  cat("LeBron James actual log salary:   ", round(lebron_actual_log, 4), "\n")
  cat("LeBron James predicted log salary:", round(lebron_pred_log,   4), "\n")
  cat("Difference (actual - predicted):  ",
      round(lebron_actual_log - lebron_pred_log, 4), "\n")

  if (lebron_actual_log > lebron_pred_log) {
    cat("Conclusion: LeBron James appears OVERPAID relative to model predictions.\n")
  } else {
    cat("Conclusion: LeBron James appears UNDERPAID relative to model predictions.\n")
  }
